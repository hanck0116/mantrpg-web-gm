# ManTRPG Development Instructions

이 프로젝트는 ManRPG_v18_FINAL_병합패키지를 기준으로 한다.

## 절대 원칙

- 이 게임은 모바일 우선 PWA 웹앱이다.
- 중앙 서버를 사용하지 않는다.
- 유료 API를 사용하지 않는다.
- 외부 DB를 사용하지 않는다.
- 기본 플레이는 AI 없이도 가능해야 한다.
- AI 기능은 선택 기능으로만 둔다.
- 모든 규칙은 JavaScript 코드로 강제한다.
- AI는 규칙 판정자가 아니라 서술자 또는 자연어 해석자 역할만 한다.
- 적은 항상 1명이다.
- 보스몹은 만들지 않는다.
- 맵은 항상 동일한 2D 격자 맵을 사용한다.
- UI는 항상 간단하고 모바일 친화적으로 만든다.

## 게임 구조

- 장르는 턴제 텍스트 TRPG다.
- 플레이어는 자연어 또는 버튼으로 행동을 선언한다.
- 전투는 턴제로 처리한다.
- 모든 판정은 스탯 기반 주사위로 처리한다.
- 주사위는 AI가 굴리지 않고 코드가 굴린다.
- HP, MP, 코인, 보상, 마법서, 상태 변화는 코드가 처리한다.

## 금지 사항

- WASD 이동 금지
- 실시간 회피 금지
- 실시간 가드 금지
- 히트박스 금지
- 마법 쿨타임 금지
- 장착 마법 버튼 금지
- 실시간 전투 애니메이션 중심 구현 금지
- 서버 필수 구조 금지
- 유료 API 필수 구조 금지
- AI가 보상을 생성하게 만들기 금지
- AI가 적 행동을 결정하게 만들기 금지
- AI가 주사위를 굴리게 만들기 금지

## 전술 맵

- 화면 하단에 현재 상태 맵을 표시한다.
- 맵에는 플레이어 위치, 적 위치, 공격 예고, 파괴된 지형, 불, 낙인, 마법 효과, 환경 변화를 표시한다.
- 맵은 2D CSS Grid로 구현한다.
- Canvas나 3D는 MVP 이후에 검토한다.

## 개발 방식

작업은 반드시 작은 단위로 진행한다.

1. 기본 파일 구조 생성
2. 상태표 구현
3. 2D 맵 구현
4. 주사위 엔진 구현
5. 전투 턴 구현
6. 적 AI 구현
7. 보상 엔진 구현
8. 심상세계 절차 구현
9. 저장/불러오기 구현
10. PWA 설정
11. 모바일 화면 최적화
