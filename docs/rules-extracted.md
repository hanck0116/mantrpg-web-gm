## 현재 코드에 반영 가능한 확정 원칙

아래 항목은 저장소의 `AGENTS.md` 및 기존 `RULES.md`에서 확인한 원칙이다. 원본 zip이 없는 상태에서는 세부 수치를 새로 추측하지 않는다.

### 프로젝트 원칙

- 모바일 우선 PWA 웹앱이다.
- 중앙 서버, 유료 API, 외부 DB를 필수로 사용하지 않는다.
- 기본 플레이는 AI 없이 가능해야 한다.
- AI 기능은 선택 기능이며 규칙 판정자가 아니다.
- 모든 규칙 판정은 JavaScript/TypeScript 코드가 처리한다.
- 주사위, HP, MP, 코인, 보상, 마법서, 상태 변화는 코드가 처리한다.

### 전투/맵 원칙

- 장르는 턴제 텍스트 TRPG다.
- 적은 항상 1명이다.
- 보스몹은 만들지 않는다.
- 맵은 항상 동일한 2D 격자 맵을 사용한다.
- MVP 맵은 7x7 고정 CSS Grid로 표시한다.
- WASD 이동, 실시간 회피, 실시간 가드, 히트박스, 실시간 전투 애니메이션 중심 구현은 금지한다.

### 기존 RULES.md에서 확인한 MVP 규칙

- 기본 스탯은 힘, 민첩, 체력, 지능, 지혜, 외모다.
- 스탯 값이 주사위 최대값이다. 예: 힘 12는 1d12다.
- 한 턴에 기본 행동 1회를 수행한다.
- 공격, 방어, 이동, 마법, 관찰, 자유 행동이 가능하다.
- 최대 HP는 `20 + 체력`이다.
- 최대 MP는 `4 + 지능`이다.
- 층 클리어 보상은 2개 생성 후 1개를 선택한다.
- 보상은 AI가 생성하지 않고 코드가 생성한다.
- 심상세계 순서는 보상 획득, 스탯 분배, 스킬 생성, 마법서 관련 행동, 상점 이용, 다음 층 진입 여부 선택이다.

## 이번 단계의 구현 범위

- Vite + TypeScript 프로젝트 뼈대를 만든다.
- 브라우저에 기본 화면을 표시한다.
- 7x7 고정 맵을 표시한다.
- 플레이어 1명과 적 1명을 표시한다.
- 기본 상태창과 전투 로그 영역을 표시한다.
- AI 설정 패널 자리만 만든다.
- AI 라우터와 Cloudflare Worker는 타입과 자리만 만들고 실제 호출은 구현하지 않는다.
